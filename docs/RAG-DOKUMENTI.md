# Ръководство: Как да добавяш информация за автомобилния асистент (RAG)

**Classic Car Center — база знания за Ollama асистента**

---

## 1. Какво прави асистентът

Преди да отговори на въпрос, асистентът:

1. **Търси** в индексирана база от текстове (документи)
2. **Избира** най-релевантните части (chunks)
3. **Подава** ги на AI модела (`qwen2.5:7b` и др.)
4. **Отговаря** предимно на база тези документи — на български или английски

Това се нарича **RAG** (Retrieval-Augmented Generation).  
Асистентът **не чете PDF, Word или снимки** — само **plain text**, който системата сама индексира.

---

## 2. Откъде идва информацията днес

| Източник | Файл / място | Какво съдържа |
|----------|--------------|---------------|
| **Vault (база данни)** | `/vault/add` в сайта | Статии за конкретни класически коли |
| **Vault seed** | `lib/classicCarsSeed.ts` | 20 предварително написани статии |
| **Шоурум** | Обяви през „Добави кола“ | Описания на коли за продажба |
| **Марки (brand KB)** | `lib/rag/brandKnowledge.ts` + `brandKnowledgeExtra.ts` | ~103 марки — ранна история, класика, модели |
| **Каталог модели** | `app/add-car/carData.ts` | Списък марки/модели (автоматично се добавя към марките) |

**Индексът** (векторно търсене) се пази локално в:
```
.rag/index.json
```
Този файл се **генерира автоматично** — не го редактираш ръчно.

---

## 3. Задължителна стъпка след всяка промяна

След като добавиш или промениш информация, **винаги преиндексирай**:

### Вариант А — от сайта
1. Отвори http://localhost:3000/ollama-test
2. Натисни **„Индексирай документи“**
3. Изчакай 2–5 минути (зависи от броя документи)

### Вариант Б — от терминала
```bash
curl -X POST http://localhost:3000/api/ollama/index
```

### Проверка на статуса
```bash
curl http://localhost:3000/api/ollama/index
```

Отговорът показва `chunkCount` (брой индексирани части) и `builtAt` (кога е създаден индексът).

> **Важно:** Без преиндексиране асистентът продължава да ползва **стария** индекс и няма да вижда новите данни.

## 4. Начин 1: Vault — най-лесният (без код)

### Кога да го ползваш
- Искаш да добавиш **конкретна класическа кола** с история, характеристики, съвети за колекционери
- Искаш да редактираш през уеб интерфейса

### Стъпки
1. Влез като админ (`test@w.bg`)
2. Отвори **Научи повече** → **Добави статия** (`/vault/add`)
3. Попълни всички полета — **всеки текст влиза в RAG**

### Кои полета са важни за асистента

| Поле | Какво да пишеш | Пример |
|------|----------------|--------|
| `name` | Пълно име на модела | Porsche 356 Speedster |
| `year` | Година | 1954 |
| `make` | Марка | Porsche |
| `country` | Държава на произход | Германия |
| `description` | Кратко описание (2–5 изречения) | „Porsche 356 Speedster е…" |
| `fullHistory` | По-дълга история | Произход, развитие, значение |
| `highlights` | 4–6 кратки точки | Ключови факти |
| `raceHistory` | Състезателна история | SCCA, рали, победи |
| `collectorsInfo` | За колекционери | matching numbers, цена, какво да се провери |
| `engine`, `horsepower`, `price`… | Технически данни | Попадат в „технически профил“ chunk |

### Формат на текста
- **Обикновен текст** — без HTML, без Word форматиране
- **Български език** — препоръчително за по-добри отговори
- **Споменавай марката и модела** в текста (не само „тя“, „този модел“)
- **Конкретни факти** — години, двигател, събития, имена

### Примерен `description`
```
Porsche 356 Speedster е една от най-чистите интерпретации на следвоенната 
европейска спортна кола. Моделът е разработен с фокус върху лекота и директно 
управление. Ниското челно стъкло и боксер двигателят създават уникално 
аналогово усещане зад волана.
```

### След запис
→ **Индексирай документи** (виж раздел 3)

## 5. Начин 2: Шоурум — обяви за продажба

### Кога да го ползваш
- Искаш асистентът да знае за **реални коли в шоурума**
- Колата е добавена през „Добави кола“

### Какво се индексира
Само обяви с **попълнено поле „Описание“**.

Индексираните данни включват:
- Марка, модел, година
- Тип, цена, пробег, локация
- Описание (най-важното за RAG)

### Добър формат на описание
```
Mercedes-Benz W123 240D, 1982 г. Втори собственик в България.
Оригинален дизелов двигател, редовна поддръжка. Без ръжда по шасито.
Подходяща за ежедневна употреба и колекция.
```

### След нова/редактирана обява
→ **Индексирай документи**

## 6. Начин 3: Seed файл — много коли наведнъж

### Кога да го ползваш
- Искаш да добавиш **много статии** директно в кода
- Работиш с Git и искаш версиониране на съдържанието

### Файл
```
lib/classicCarsSeed.ts
```

### Как да добавиш нова кола

Намери масива `CLASSIC_CARS_SEED` и добави нов обект:

```typescript
{
  id: 21,  // уникален номер
  name: "BMW 2002",
  year: 1973,
  country: "Германия",
  make: "BMW",
  era: "1970s",
  type: "Sedan",
  produced: 400000,
  engine: "2.0L Inline-4",
  horsepower: "100",
  topSpeed: 175,
  price: "$40,000+",
  rarity: "Uncommon",
  image: "https://...",
  description: "Кратко описание на български...",
  fullHistory: "По-дълга история на български...",
  highlights: [
    "Първа ключова точка",
    "Втора ключова точка",
  ],
  designer: "…",
  weight: "1020 kg",
  raceHistory: "Състезателна история...",
  collectorsInfo: "Информация за колекционери...",
},
```

### Важно за дубликати
Ако същата кола (`name` + `year`) вече съществува в **базата данни** (Vault), seed версията се **пропуска** — приоритет има базата.

### След промяна
```bash
npm run dev   # рестарт не е задължителен, но препоръчителен
```
→ **Индексирай документи**

## 7. Начин 4: Марки — история, начало, класика

### Кога да го ползваш
- Въпроси от типа: *„Какво произвеждаше Škoda в началото?“*
- *„Как започва BMW?“*, *„Какви са ранните модели на Lada?“*
- Обща информация за **марка**, не за един конкретен модел

### Файлове

| Файл | Съдържание |
|------|------------|
| `lib/rag/brandKnowledge.ts` | Основни 39 марки |
| `lib/rag/brandKnowledgeExtra.ts` | Още 64 марки (Lada, Moskvich, Trabant, Rolls-Royce…) |
| `lib/rag/brandTypes.ts` | Тип на данните (не редактирай освен ако знаеш TypeScript) |

### Как да добавиш нова марка

Отвори `lib/rag/brandKnowledgeExtra.ts` и добави в масива:

```typescript
{
  make: "Lada",                    // официално име на марката
  aliases: [                       // как потребителят може да я напише
    "лада", "Лада", "ваз", "ВАЗ", "жигули", "Жигули"
  ],
  country: "Русия/СССР",
  founded: "1966 (VAZ, Жигули)",
  earlyHistory:                    // НАЙ-ВАЖНО за въпроси „в началото“
    "Lada (ВАЗ/Жигули) започва през 1966 г. в Толиати с лиценз на Fiat 124…",
  classicEra:                      // класически/известни модели
    "Класически Lada: 2101, 2106, Niva, Samara…",
  heritage:                        // общо значение, днес
    "В България Жигули и Нива са изключително разпространени…",
  models: [                        // по избор — списък модели
    "2101", "2106", "Niva", "Samara", "Vesta"
  ],
},
```

### Обяснение на полетата

| Поле | Защо е важно |
|------|--------------|
| `make` | Име на марката в индекса |
| `aliases` | Български/латиница варианти — подобрява търсенето („шкода“, „škoda“) |
| `earlyHistory` | Отговаря на „какво произвеждаше в началото“, „ранна история“ |
| `classicEra` | Отговаря на „класически модели“, „колекционерски“ |
| `heritage` | Общ контекст и значение |
| `models` | Списък модели (ако марката липсва в `carData.ts`) |

### Как се разделя на части (автоматично)

Всяка марка става **3–4 chunks**:
1. `overview` — общ преглед
2. `early-history` — **ранна история** ← ключово за „в началото“
3. `classic-era` — класически модели
4. `models` — списък модели (от `carData.ts` или `models` поле)

### След промяна
→ **Индексирай документи**

## 8. Препоръки за качество на текста

### Прави това
- Пиши на **грамотен български**
- Използвай **конкретни години, имена, модели**
- Прави **кратки параграфи** (3–6 изречения)
- Включвай **автомобилна терминология**: шаси, двигател, каросерия, matching numbers, реставрация
- За марки — опиши **първите продукти** (велосипеди, мотоциклети, първи автомобил)

### Избягвай
- Празни полета
- Само едно-две думи без контекст
- HTML тагове (`<p>`, `<br>`)
- Копиране от PDF с счупено форматиране
- Твърде общи фрази без факти („много добра кола“)

### Примери за добри въпроси, които ще работят след добавяне

| Въпрос | Къде да е информацията |
|--------|------------------------|
| Какво произвеждаше Škoda в началото? | `brandKnowledge` → `earlyHistory` |
| Какво да проверя при покупка на Porsche 356? | Vault статия → `collectorsInfo` |
| Има ли Jaguar E-Type в базата? | Vault seed или DB |
| Какви коли има в шоурума? | Обяви с описание |

## 9. Какво НЕ се поддържа (засега)

| Формат | Статус |
|--------|--------|
| PDF | ❌ Не се чете автоматично |
| Word (.docx) | ❌ |
| Excel | ❌ |
| Снимки/сканирани документи | ❌ |
| `.md` / `.txt` файлове в папка | ❌ (трябва код за зареждане) |
| Уебсайтове / Wikipedia | ❌ |

**Ако имаш PDF или Word:** копирай текста в Vault форма или в `brandKnowledgeExtra.ts`.

## 10. Архитектура (за разработчици)

```
Потребителски въпрос
        ↓
app/api/ollama/route.ts
        ↓
lib/rag/index.ts → searchCarDocuments()
        ↓
lib/rag/embeddings.ts → nomic-embed-text (Ollama)
        ↓
.rag/index.json (локален векторен индекс)
        ↓
Топ 5 релевантни chunks → подават се на qwen2.5:7b
        ↓
Отговор на български/английски
```

### Ключови файлове

| Файл | Роля |
|------|------|
| `lib/rag/documents.ts` | Събира всички източници в един списък chunks |
| `lib/rag/brandChunks.ts` | Превръща марките в chunks |
| `lib/rag/brandSearch.ts` | Подобрява търсенето по име на марка |
| `lib/rag/index.ts` | Индексиране и търсене |
| `app/api/ollama/index/route.ts` | API за преиндексиране |
| `hooks/useCarAssistant.ts` | UI логика на асистента |

### Типове източници (`source`)

| source | Значение |
|--------|----------|
| `vault-db` | Статии от базата данни |
| `vault-seed` | Статии от `classicCarsSeed.ts` |
| `showroom` | Обяви от шоурума |
| `brand-kb` | Марки от brand knowledge |

## 11. Чести проблеми

| Проблем | Причина | Решение |
|---------|---------|---------|
| Асистентът не знае новата информация | Не е преиндексирано | „Индексирай документи“ |
| Грешен отговор за марка | Липсва марка в brand KB | Добави в `brandKnowledgeExtra.ts` |
| Отговор на английски при български въпрос | Грешен chat модел | Използвай `qwen2.5:7b`, не `coder` |
| „Document search failed“ | Липсва embedding модел | `ollama pull nomic-embed-text` |
| „Ollama chat failed“ + embed модел | Избран `nomic-embed-text` за чат | Избери `qwen2.5:7b` в Chat модел |
| Дублирани статии в индекса | Seed + DB с еднаква кола | Нормално — DB има приоритет |

## 13. Пример: пълен workflow за нова марка „Wartburg“

1. Отвори `lib/rag/brandKnowledgeExtra.ts`
2. Добави обект за Wartburg (или редактирай съществуващия)
3. Попълни `earlyHistory` с факти за Eisenach, 311, 353, тритактов двигател
4. Добави aliases: `["варбург", "Вартбург", "wartburg"]`
5. Запази файла
6. Отвори `/ollama-test` → **Индексирай документи**
7. Тествай: *„Какво произвеждаше Wartburg в началото?“*

---

## 14. Пример: пълен workflow за нова класическа кола

1. Влез като админ → `/vault/add`
2. Попълни всички полета за колата (на български)
3. Запази статията
4. **Индексирай документи**
5. Тествай: *„Защо [име на колата] е ценена от колекционери?“*
6. Провери „Източници“ под отговора в асистента
